# Importing Data

WhoDB's current import flow lives inside the table view and supports two paths:

- structured file import for CSV and Excel
- SQL import from a pasted script or `.sql` file

## Open The Import Flow

Open a table, then use the **Import** button in the grid's bottom footer area, next to **Export All**, or press `Mod + I`.

<Note>
Structured file imports target the table you currently have open.
</Note>

## CSV And Excel Import

<Steps>
<Step title="Choose The File Type">
The current UI supports:

- `.csv`
- `.xlsx`
- `.xlsm`
</Step>
<Step title="Use Headers">
For in-app CSV and Excel imports, the first row is treated as the header row.

Match those headers to the target table's column names. If the file includes an auto-generated column such as an ID, WhoDB shows a checkbox so you can explicitly allow that column.
</Step>
<Step title="Review The Preview">
WhoDB loads a preview before import and shows:

- detected columns
- up to three sample rows
- validation messages when the file and table do not line up
- detected Excel sheet name when applicable
</Step>
<Step title="Adjust File Options">
The current import panel supports:

- automatic or manual CSV delimiter selection
- automatic Excel sheet detection
- append, overwrite, and upsert import modes
- a toggle for allowing auto-generated columns when required by validation
</Step>
<Step title="Run The Import">
Once the preview is valid, submit the import to write the rows into the current table.
</Step>
</Steps>

## SQL Import

SQL import can start from either:

- a pasted SQL script
- an uploaded SQL file

Use one source at a time: either paste SQL or upload a file. The current flow requires an explicit confirmation checkbox before execution.

## Current Limits

The import panel uses these limits:

- maximum file size: `50 MB`
- maximum row count: `100,000`
- preview rows shown in the UI: `3`

## Validation Behavior

WhoDB stops the import and shows a validation message for issues such as:

- invalid or missing column mapping
- auto-generated columns that require explicit consent
- parser or file-format errors

Use the preview result to fix the file or adjust the import mode before retrying.

## Related Tools

If you need scriptable or terminal-based CSV and Excel imports, use the CLI:

```bash
whodb-cli import --connection mydb --file data.csv --table users
```

Useful CLI flags include:

- `--schema`
- `--delimiter`
- `--sheet`
- `--header=false`
- `--mapping position`
- `--mode append`
- `--mode overwrite`
- `--mode upsert`
- `--create-table`
- `--allow-auto-generated`

For scriptable SQL imports, use the native database client for your engine, such as `psql -f seed.sql` or `mysql database < seed.sql`.

See [CLI & MCP](/features/cli-and-mcp) for the current command surface.
