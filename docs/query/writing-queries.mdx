# Writing Queries

Scratchpad uses a SQL code editor for custom database queries. It supports SQL syntax highlighting, statement execution from the editor, query history, and result rendering below the cell.

![Scratchpad Code Editor](/images/28-scratchpad-code-editor.png)

## Basic Query Flow

<Steps>
<Step title="Open Scratchpad">
Use the sidebar **Scratchpad** item.
</Step>
<Step title="Write SQL">
Type a query in the active cell.
</Step>
<Step title="Run">
Click the run button at the bottom-right of the cell (it appears on hover and is hidden while the editor is empty), use a per-statement play button in the editor's left gutter, or press `Cmd/Ctrl + Enter`.
</Step>
<Step title="Review Output">
Results, action status, or errors appear below the editor.
</Step>
</Steps>

```sql
SELECT *
FROM users
LIMIT 10;
```

## Running Part Of A Query

If you select SQL text in the editor and press `Cmd/Ctrl + Enter`, WhoDB runs the selected SQL. If nothing is selected, it runs the editor content.

This is useful for keeping related statements in one cell while testing only one statement at a time.

## SELECT Queries

`SELECT` queries return an interactive grid.

![Select Query Result](/images/86-scratchpad-select-query-result.png)

```sql
SELECT id, username, email
FROM users
ORDER BY id
LIMIT 25;
```

## Aggregate Queries

Aggregate queries work like any other SQL your database accepts.

![Count Query Result](/images/87-scratchpad-count-query-result.png)

```sql
SELECT COUNT(*) AS total_users
FROM users;
```

## Joins

Use normal SQL joins for your database engine.

![Join Query Result](/images/88-scratchpad-join-query-result.png)

```sql
SELECT u.id, u.username, p.amount
FROM users u
JOIN payments p ON p.user_id = u.id
ORDER BY p.amount DESC
LIMIT 25;
```

## Action Queries

Action queries such as `INSERT`, `UPDATE`, and `DELETE` are supported when your database credentials allow them.

```sql
UPDATE users
SET status = 'inactive'
WHERE id = 42;
```

WhoDB detects destructive SQL and asks for confirmation before execution.

![Scratchpad Action Result](/images/40-scratchpad-action-result.png)

## Errors

Syntax errors and database runtime errors are displayed below the editor.

![Query Error](/images/30-scratchpad-query-error.png)

Use a smaller query to isolate the failure:

```sql
SELECT 1;
```

Then add tables, joins, filters, and ordering back one step at a time.

## Current Shortcuts

| Shortcut | Action |
|----------|--------|
| `Cmd/Ctrl + Enter` | Run current query or selected SQL |
| `Cmd/Ctrl + U` | Clear current editor |
