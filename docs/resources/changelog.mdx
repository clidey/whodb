# WhoDB Changelog

Stay informed about the latest features, improvements, and bug fixes in WhoDB. This changelog documents all significant updates across Community Edition (CE) and Enterprise Edition (EE).

<Tip>
Subscribe to GitHub releases for instant notifications of new WhoDB versions
</Tip>

## v1.5.0 (Latest Release)

**Release Date:** Q4 2024

### New Features

- **AI Chat Enhancements**: Improved schema awareness and multi-table query generation
- **Keyboard Shortcut Customization**: Configure custom keyboard shortcuts in settings
- **Advanced Export Options**: New export presets and quick-export templates
- **Table Comparison Tool**: Compare schema and data between similar tables
- **Saved Queries Feature**: Save frequently used queries in organized collections
- **Dark Mode Enhancement**: Improved visual hierarchy and contrast ratios

### Improvements

- Performance optimization for large dataset pagination (50% faster)
- Enhanced Scratchpad multi-cell workflow with better cell organization
- Improved Graph visualization for schemas with 100+ tables
- Better error messages with actionable solutions
- Faster database connection establishment
- Reduced memory footprint on frontend by 30%
- Improved accessibility with better keyboard navigation
- Enhanced search functionality with fuzzy matching

### Bug Fixes

- Fixed data grid column header overflow issues
- Resolved Scratchpad query result caching problems
- Fixed Graph view zoom level persistence
- Corrected WHERE condition operator behavior for NULL values
- Fixed export encoding issues with special characters
- Resolved connection profile import errors
- Fixed sidebar collapse state not persisting
- Corrected mock data generation for specific PostgreSQL types

### Database Support Updates

- **PostgreSQL**: Added support for v16 with improved JSON functions
- **MongoDB**: Enhanced document preview with better formatting
- **ClickHouse**: Added support for dictionary tables and functions
- **Elasticsearch**: Improved index mapping visualization

### Known Issues

- EE: Oracle Database connection pooling requires manual configuration
- Large Excel exports (10k+ rows) may take longer than CSV
- Graph visualization performance on schemas with 500+ tables

## v1.0.0 (Initial Release)

**Release Date:** Q2 2023

### Features

- **Multi-Database Support**: Connect to PostgreSQL, MySQL, SQLite, MongoDB
- **Data Grid View**: View and navigate table data
- **Sidebar Navigation**: Browse database structure
- **Connection Management**: Save and switch between database connections
- **Browser-Based Interface**: No installation required (with Docker)
- **Lightweight Design**: Minimal resource usage

### Database Support

- PostgreSQL 9.6+
- MySQL 5.7+
- SQLite 3.8+
- MongoDB 3.6+

### Known Limitations (Fixed in Later Versions)

- Limited export capabilities (JSON only)
- No query editor (added in v1.2.0)
- No schema visualization (added in v1.3.0)
- Limited keyboard shortcuts (expanded in v1.4.0+)

## Release Process

WhoDB follows semantic versioning (MAJOR.MINOR.PATCH):

- **Major Version**: Significant features, breaking changes, or major improvements
- **Minor Version**: New features and improvements without breaking changes
- **Patch Version**: Bug fixes and security updates

## Breaking Changes History

<AccordionGroup>
<Accordion title="v1.0 → v1.1">
- No breaking changes
- Full backward compatibility maintained
</Accordion>

<Accordion title="v1.1 → v1.2">
- Query history storage format changed
- Old history entries may need to be re-saved
- No API breaking changes
</Accordion>

<Accordion title="v1.2 → v1.3">
- Keyboard shortcuts restructured
- Custom shortcuts will be reset to defaults
- No database compatibility changes
</Accordion>

<Accordion title="v1.3 → v1.4">
- ChatDB/Ollama parameter format updated
- Old configurations will auto-migrate
- Connection profile format enhanced but backward compatible
</Accordion>

<Accordion title="v1.4 → v1.5">
- No breaking changes
- Full backward compatibility with v1.4 profiles
- All features remain compatible
</Accordion>
</AccordionGroup>
