# Keyboard Shortcuts Reference

Speed up your WhoDB workflow with these keyboard shortcuts. Whether you're navigating databases, writing queries, or editing data, mastering these shortcuts will make you significantly more productive.

<Tip>
Bookmark this page for quick reference while working in WhoDB
</Tip>

## Global Shortcuts

These shortcuts work throughout WhoDB and are accessible from any view or context.

| Shortcut | Action | Notes |
|----------|--------|-------|
| `Cmd/Ctrl + K` | Open command palette | Quickly navigate to any feature or database |
| `Cmd/Ctrl + /` | Toggle sidebar | Show or hide the left navigation panel |
| `Cmd/Ctrl + F` | Find in current view | Search text on the current page |
| `Esc` | Close dialogs/modals | Exit any open dialog or context menu |
| `Cmd/Ctrl + Shift + ?` | Show keyboard shortcuts help | Display all available shortcuts |
| `Tab` | Focus next element | Navigate between interactive elements |
| `Shift + Tab` | Focus previous element | Navigate backwards between elements |
| `Enter` | Confirm/Submit | Accept dialog or complete action |
| `Cmd/Ctrl + ,` | Open settings | Access application preferences |
| `Cmd/Ctrl + Alt + D` | Toggle dark mode | Switch between light and dark theme |

## Data Grid Shortcuts

Use these shortcuts when viewing table data to navigate and edit records efficiently.

| Shortcut | Action | Notes |
|----------|--------|-------|
| `Cmd/Ctrl + A` | Select all rows | Select every row in the current table or filtered view |
| `Cmd/Ctrl + C` | Copy selected cell | Copy cell value to clipboard |
| `Cmd/Ctrl + X` | Cut selected cell | Cut and copy cell value |
| `Cmd/Ctrl + V` | Paste to cell | Paste clipboard content into cell |
| `Tab` | Move to next cell | Navigate right or to first cell in next row |
| `Shift + Tab` | Move to previous cell | Navigate left or to last cell in previous row |
| `Arrow Up` | Move up one row | Navigate to cell above current |
| `Arrow Down` | Move down one row | Navigate to cell below current |
| `Arrow Left` | Move left one column | Navigate to cell on the left |
| `Arrow Right` | Move right one column | Navigate to cell on the right |
| `Enter` | Edit selected row | Open edit dialog for current row |
| `Double-click` | Inline edit cell | Edit cell value directly in the grid |
| `Escape` | Cancel editing | Discard changes and exit edit mode |
| `Delete` | Clear cell value | Empty the current cell (in edit mode) |
| `Cmd/Ctrl + D` | Duplicate row | Create a copy of the current row |
| `Cmd/Ctrl + Shift + A` | Add new row | Open dialog to add a new record |
| `Cmd/Ctrl + Shift + Delete` | Delete row | Remove the current row (with confirmation) |
| `Page Down` | Scroll down | Load and view next page of results |
| `Page Up` | Scroll up | Load and view previous page of results |
| `Home` | Go to first cell | Jump to first column of current row |
| `End` | Go to last cell | Jump to last column of current row |
| `Cmd/Ctrl + Home` | Go to first cell | Jump to top-left corner of table |
| `Cmd/Ctrl + End` | Go to last cell | Jump to bottom-right corner of table |

## Scratchpad Query Editor Shortcuts

The Scratchpad is WhoDB's powerful query interface. Use these shortcuts to write and execute queries faster.

| Shortcut | Action | Notes |
|----------|--------|-------|
| `Cmd/Ctrl + Enter` | Execute query | Run the current cell's SQL query |
| `Cmd/Ctrl + S` | Save query | Save query to history |
| `Cmd/Ctrl + N` | New cell | Insert a new query cell below current |
| `Cmd/Ctrl + D` | Duplicate cell | Create a copy of the current cell |
| `Cmd/Ctrl + Shift + Delete` | Delete cell | Remove the current query cell |
| `Cmd/Ctrl + M` | Merge cells | Combine current and next cell |
| `Cmd/Ctrl + /` | Toggle comment | Comment or uncomment selected lines |
| `Cmd/Ctrl + Shift + F` | Format query | Auto-format SQL code with proper indentation |
| `Cmd/Ctrl + F` | Find in editor | Search within query text |
| `Cmd/Ctrl + H` | Find and replace | Search and replace text in queries |
| `Cmd/Ctrl + Z` | Undo | Revert last change in editor |
| `Cmd/Ctrl + Shift + Z` | Redo | Reapply last undone change |
| `Tab` | Insert indentation | Add spaces for code formatting |
| `Shift + Tab` | Remove indentation | Decrease indentation level |
| `Cmd/Ctrl + A` | Select all | Select all text in current cell |
| `Cmd/Ctrl + [` | Decrease font size | Make code smaller (if supported) |
| `Cmd/Ctrl + ]` | Increase font size | Make code larger (if supported) |
| `Up Arrow` | Previous query in history | Load previous query from history |
| `Down Arrow` | Next query in history | Load next query from history |
| `Cmd/Ctrl + Shift + H` | Open query history | Display full query history panel |
| `Escape` | Close autocomplete | Hide code suggestions dropdown |

Quickly navigate between different views and database objects.

|----------|--------|-------|
| `Cmd/Ctrl + 1` | Go to Data view | Switch to table data view |
| `Cmd/Ctrl + 2` | Go to Explore view | Switch to schema explorer |
| `Cmd/Ctrl + 3` | Go to Scratchpad view | Switch to query editor |

| `Cmd/Ctrl + L` | Focus on table selector | Jump to select different table |
| `Cmd/Ctrl + Shift + L` | Focus on database selector | Jump to select different database |
| `Cmd/Ctrl + Shift + S` | Focus on schema selector | Jump to select different schema |
| `Alt + 1` | First open tab | Jump to first tab |
| `Alt + 2` | Second open tab | Jump to second tab |
| `Alt + N` | Nth open tab | Jump to Nth tab |
| `Cmd/Ctrl + W` | Close current tab | Close active view or connection |

## Platform-Specific Notes

### macOS

- Use `Cmd` instead of `Ctrl` for all shortcuts
- `Option` is the equivalent of `Alt`
- Keyboard shortcuts work in most applications

### Windows & Linux

- Use `Ctrl` instead of `Cmd`
- `Alt` key available for Windows shortcuts
- Modifier combinations may vary by application

### Mobile & Tablet

- Most keyboard shortcuts not available
- Touch gestures replace keyboard actions
- On-screen keyboard may be limited

## Customizing Keyboard Shortcuts

You can customize keyboard shortcuts in the settings:

<Steps>
<Step title="Open Settings">
Press `Cmd/Ctrl + ,` to open the settings panel
</Step>
<Step title="Navigate to Shortcuts">
Click on "Keyboard Shortcuts" in the settings menu
</Step>
<Step title="Search for Command">
Find the shortcut you want to customize
</Step>
<Step title="Edit Shortcut">
Click the shortcut and enter your preferred key combination
</Step>
<Step title="Save Settings">
Changes are saved automatically
</Step>
</Steps>

<Note>
Some shortcuts cannot be customized as they conflict with browser or OS shortcuts
</Note>

## Tips for Keyboard Power Users

<AccordionGroup>
<Accordion title="Reduce Mouse Usage">
- Use `Tab` and `Shift + Tab` to navigate between elements
- Use `Enter` and `Space` to activate buttons
- Use arrow keys for precise navigation
- Combine with `Cmd/Ctrl` for faster jumps
</Accordion>

<Accordion title="Workflow Automation">
- Combine shortcuts to create efficient workflows
- Example: `Cmd/Ctrl + F` then `Cmd/Ctrl + Shift + F` for advanced search
- Use history shortcuts to repeat common queries
- Save frequently used shortcuts in your muscle memory
</Accordion>

<Accordion title="Productivity Tips">
- Use `Cmd/Ctrl + K` for quick navigation to any feature
- Memorize the numeric shortcuts (`Cmd/Ctrl + 1` through `5`) for view switching
- Master `Cmd/Ctrl + Enter` for fast query execution
- Use `Cmd/Ctrl + N` and `Cmd/Ctrl + D` to quickly organize Scratchpad
</Accordion>

<Accordion title="Common Shortcut Combinations">
**Quick Data Export Workflow:**
1. `Cmd/Ctrl + A` - Select all rows
2. `Cmd/Ctrl + Shift + E` - Export as CSV

**Multi-Cell Query Workflow:**
1. `Cmd/Ctrl + N` - New cell
2. `Cmd/Ctrl + Enter` - Execute
3. `Cmd/Ctrl + D` - Duplicate for similar query

**Navigation Workflow:**
1. `Cmd/Ctrl + K` - Open command palette
2. Type table name
3. `Enter` - Navigate to table
</Accordion>
</AccordionGroup>

## Learning Resources

<AccordionGroup>
<Accordion title="Getting Started with Shortcuts">
- Start with global shortcuts (`Cmd/Ctrl + K`, `Cmd/Ctrl + /`)
- Move to view-specific shortcuts based on your workflow
- Gradually add more shortcuts as they become useful
- Focus on the most frequently used actions
</Accordion>

<Accordion title="Building Muscle Memory">
- Practice one shortcut at a time
- Use the shortcut repeatedly until it becomes automatic
- Keep this reference page open while learning
- Test yourself periodically
</Accordion>

<Accordion title="Troubleshooting Shortcuts">
If a shortcut doesn't work:
- Check if your OS or browser has intercepted it
- Try the equivalent for your platform (Cmd vs Ctrl)
- Reset to default shortcuts in settings
- Check the settings panel for custom assignments
</Accordion>
</AccordionGroup>
